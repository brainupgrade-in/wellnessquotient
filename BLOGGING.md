# Publishing a blog post — operation guide

This is the **single source of truth** for adding a blog post to Wellness Quotient,
written so an AI agent (or a human) can do it reliably and safely.

> **The golden rule:** To publish a post, you create **one Markdown file** in `_posts/`.
> Nothing else needs editing. The sitemap, the RSS feed, the search index, the
> homepage "latest" list, the sidebar counts, and pillar navigation **all update
> automatically** on the next build. Do **not** hand-edit `sitemap.xml` — it does
> not exist as a file; it is generated by `jekyll-sitemap` at build time.

---

## 1. Create the file

**Location:** `_posts/`
**Filename:** `YYYY-MM-DD-url-slug.md` (the date and slug become the post URL)

Example: `_posts/2026-06-15-the-architecture-of-calm.md`
→ publishes at `/blog/2026/06/the-architecture-of-calm/`

## 2. Use this front-matter template

Copy this block verbatim and fill it in. Required fields are marked.

```markdown
---
layout: post                          # required — always "post"
title: "Your Essay Title"             # required
subtitle: "One elegant sentence."     # optional, shown under the title
date: 2026-06-15 09:00:00 +0530       # required — must not be in the future, IST offset
author: Dr. Ujwala                    # optional, defaults to Dr. Ujwala
categories: [Sleep]                   # required — ONE pillar (see list below)
tags: [sleep, recovery]               # optional — lowercase keywords for search/filter
image: https://images.unsplash.com/photo-XXXX?auto=format&fit=crop&w=1400&q=70
image_alt: "Describe the image"       # optional but recommended for accessibility/SEO
image_caption: "A short caption."     # optional
excerpt: "2–3 sentence teaser shown on cards and in search results."  # recommended
---

Your essay body in Markdown starts here…
```

## 3. Choose the right pillar (the `categories` value)

Use **exactly one** of these, capitalised as shown. This drives the sidebar
filter, the pillar badge, and the footer navigation:

`Food` · `Stress` · `Exercise` · `Belongingness` · `Sleep` · `Environment` · `Longevity`

> Adding a new pillar? Edit the `pillars:` list in `_config.yml` — the sidebar,
> footer, and homepage grid all read from there.

## 4. Writing conventions (house style)

- **Audience:** discerning, affluent, time-poor readers. Evidence over hype, calm over urgency.
- **Voice:** considered, literate, unhurried. Short paragraphs. One pull-quote per essay using `>`.
- **Length:** roughly 600–1,100 words. Reading time is computed automatically.
- **Structure:** open with a hook, use `##` section headings, close with a takeaway.
- **No medical claims as advice** — this is editorial guidance, not treatment.

## 5. Images

- Prefer a wide (≈1400px) image. Unsplash CDN URLs work directly (see existing posts),
  or commit a file to `assets/images/` and reference it as `/assets/images/name.jpg`.
- Always set `image_alt` for accessibility and SEO.
- If you omit `image`, the card and post simply render without one — that's fine.

## 6. Publish

```bash
git add _posts/2026-06-15-the-architecture-of-calm.md
git commit -m "Add essay: The Architecture of Calm"
git push
```

GitHub Pages rebuilds within a minute or two. The new post then appears on the
homepage, in the Journal, in search, in `feed.xml`, and in `sitemap.xml`.

## 7. After publishing — Google indexing (optional, recommended)

The sitemap at `/sitemap.xml` lists every post automatically. To get a new post
indexed faster, open **Google Search Console → URL Inspection**, paste the post
URL, and click **Request Indexing**. You never need to edit the sitemap by hand.

---

## Checklist for the AI agent

- [ ] Filename is `_posts/YYYY-MM-DD-slug.md`, date not in the future
- [ ] `layout: post`, `title`, `date`, and one valid `categories` pillar are set
- [ ] `excerpt` written (2–3 sentences)
- [ ] `image_alt` set if an `image` is used
- [ ] Body uses `##` headings and one `>` pull-quote
- [ ] Committed and pushed — **no other files touched**
