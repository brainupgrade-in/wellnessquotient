# Wellness Quotient — Journal

A GitHub Pages (Jekyll) site that hosts the **Wellness Quotient** blog: a private
journal of preventive health, longevity and considered living, designed for a
discerning, high-net-worth readership.

**Live:** https://brainupgrade-in.github.io/wellnessquotient/
**Contact:** Dr. Ujwala — [t.me/DrUjwala](https://t.me/DrUjwala)

---

## What this gives you

- **Reusable templates** — header, footer, left-hand content navigation and the
  main content area are all `_includes`/`_layouts`, edited in one place.
- **Auto sitemap.xml** — generated by `jekyll-sitemap` on every build, ready for
  Google Search Console. Never edited by hand.
- **Search & navigation** — client-side keyword search and one-click pillar
  filtering on the Journal index (`assets/js/search.js` over `/search.json`).
- **Luxury healthcare aesthetic** — forest-green / champagne-gold / ivory palette,
  Playfair Display + Inter typography, editorial whitespace.
- **One-file publishing** — an AI agent adds a post by creating a single Markdown
  file. See **[BLOGGING.md](BLOGGING.md)**.

## Structure

```
_config.yml            Site config, the six pillars, plugins
_layouts/              default · page · post  (the main content area templates)
_includes/             head · header · footer · sidebar · post-card  (reusable parts)
_posts/                Blog posts (one Markdown file each)
assets/css/style.css   The theme
assets/js/search.js    Search + pillar filtering
assets/images/         Favicon and post images
index.html             Landing page
blog.html              Journal index (search + sidebar + grid)
about.html             About / Dr. Ujwala
search.json            Generated search index (Jekyll template)
robots.txt             Points crawlers at the sitemap
sitemap.xml            Auto-generated by jekyll-sitemap (not a checked-in file)
```

## Run locally

```bash
bundle install
bundle exec jekyll serve --livereload
# → http://localhost:4000/wellnessquotient/
```

## Deploy

1. Push to the `main` branch of `brainupgrade-in/wellnessquotient`.
2. In **Settings → Pages**, set **Source: Deploy from a branch**, branch `main`, folder `/ (root)`.
3. GitHub builds the Jekyll site automatically (all plugins used are Pages-whitelisted).

### Moving to a custom domain (e.g. wellnessquotient.community)

1. In `_config.yml`, set `url:` to `https://www.wellnessquotient.community` and `baseurl:` to `""`.
2. Add a `CNAME` file at the repo root containing the domain.
3. Configure DNS and enable the domain under Settings → Pages.

The sitemap, canonical URLs and feed all pick up the new domain automatically.
